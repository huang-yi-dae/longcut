## Context vs Zustand
- Zustand：create() 直接创建，import 就能用，类似全局变量
- Context：先 createContext，再用 Provider 包裹，只有内部组件能用 useAuth()
- AuthProvider 在 layout.tsx 里包裹整个应用

## Zustand 用法细节
- create((set) => ({ 数据 + 修改函数 }))
- 两种读取方式：选择器模式 useStore(s => s.xxx) 和解构模式 useStore()
- state 参数是箭头函数的形参，类似 self，名字随意

## Middleware（中间件）
- 每次用户访问页面，先执行 middleware，再加载页面
- matcher 配置哪些路由需要经过中间件
- 作用：刷新 session + 加安全头（CSP）
- 在服务器端执行，React 还没开始工作

## React / Next.js / TypeScript 关系
- TypeScript = 有类型的 JavaScript（语言）
- React = UI 库（只管组件）
- Next.js = 全栈框架（React + 路由 + 后端 + 中间件）
- .ts = 纯逻辑代码，.tsx = 包含 JSX（有标签）

## JSX 本质
- JSX = 函数调用的语法糖
- <Button onClick={fn}>点我</Button> → Button({ onClick: fn, children: "点我" })
- 编译后是 React.createElement 调用，不是 HTML

## Supabase 客户端
- client.ts（浏览器端）：用 localStorage 存 token
- server.ts（服务器端）：用 Cookie 存 token
- admin.ts（服务角色）：用 SERVICE_ROLE_KEY，绕过所有安全策略
- anon key 公开也没事，真正保护数据的是 Row Level Security

## 三种 Key
- anon key：公开，受安全策略限制
- service role key：保密，绕过安全策略
- 用户 token：登录后生成，只能操作自己的数据

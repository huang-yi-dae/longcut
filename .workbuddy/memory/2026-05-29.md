# 2026-05-29

## Stage 7.2 — 收藏视频功能设计

- 完整流程：前端 POST → Server Action → Prisma 写数据库
- 纠错：后端不用 JSX，用 ORM（Prisma）
- Server Action vs route.ts：写数据用 Server Action，读数据用 route.ts
- 收藏 = 多对多关系，关联表只需 userId + videoId
- 安全原则：不信任前端传 userId，后端从 Token 解析
- toggle 逻辑：findFirst → 有则 delete，无则 create
- 初始化：route.ts GET 查询收藏状态，设置按钮初始值

## Stage 7.3 — 乐观更新（Optimistic Update）

- 核心：先改 UI → 后台发请求 → 失败撤回
- 适合：点赞/收藏（可无损撤回）
- 不适合：删除数据、支付/注册（失败回不去）
- 判断标准：失败能否无损撤回

## Stage 7.4 — Error Boundary

- 组件渲染错误 → 整个页面白屏
- try...catch 兜不住 JSX 渲染（React 内部调用）
- Error Boundary = 渲染错误的 try...catch
- 按模块/区块包裹，炸哪块哪块显示 fallback

## Stage 7.5 — Middleware

- 全局认证关卡，页面加载前拦截
- matcher 控制哪些路径生效
- middleware.ts 放在项目根目录，一个文件管所有
- 与 withSecurity 对比：局部 vs 全局

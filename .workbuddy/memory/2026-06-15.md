# 2026-06-15 学习日志

## 今日知识点

### 1. Zustand 复习
- `create<PlayAllState>((set) => ({...}))` 创建 store
- `set({ isPlayingAll: true })` 更新状态，自动通知所有用了此 store 的组件重渲染
- `set((state) => ({ playAllIndex: state.playAllIndex + 1 }))` 新值依赖旧值时用函数形式，保证拿到最新值
- 不需要 Provider 包裹，任意组件 import 即可用

### 2. Supabase CRUD 操作（notes/route.ts）
- `.insert({...})` 新增记录，默认不返回数据
- `.insert({...}).select().single()` 新增后返回插入的记录（获取自动生成的 id、created_at 等）
- `.select('*')` 查询所有字段
- `.eq('user_id', uid)` 筛选条件（=），链式多个 eq 是 AND 关系
- `.order('created_at', { ascending: false })` 降序排序
- `.limit(n)` 限制返回条数
- `.delete().eq('id', id).eq('user_id', uid)` 删除，带双重条件
- `.upsert({...}, { onConflict: 'col1,col2' })` 存在则更新，不存在则插入（收藏功能用）

### 3. 后端权限检查
- 先取用户：`const { data: { user } } = await supabase.auth.getUser()`
- 所有操作带上 `.eq('user_id', user.id)` 防止越权
- 删笔记同时检查 noteId 和 userId：只靠 noteId 删别人数据不安全

### 4. Zod 运行时验证
- `.parse(body)` 验证失败抛出异常，需 try-catch
- `.safeParse(body)` 验证失败返回 `{ success: false, error }`，不抛异常
- `.refine(data => condition, { message })` 自定义跨字段规则（如两个可选字段至少传一个）
- `.transform(val => val.trim())` 验证通过后转换数据
- `z.string().min(1).max(500)` 链式校验长度、格式
- 先 transform 后 refine：去完空格再判断后续规则

### 5. Supabase 联表查询（JOIN）（notes/all/route.ts）
- `.select('*, video_analyses!inner(youtube_id, title, ...)')`
- `!inner` = INNER JOIN：关联表必须存在才返回结果
- 不加 `!inner` 或不写就是 LEFT JOIN：关联表不存在时字段为 null

### 6. 格式桥接（mapNote / mapNoteWithVideo）
- 数据库 snake_case（user_id, video_analyses）→ JS camelCase（userId, video）
- 每个 API route 都有对应的 map 函数做转换

### 7. Security 中间件
- `withSecurity(handler, { requireAuth, rateLimit, maxBodySize, csrfProtection })`
- rateLimit: `{ windowMs: 60*1000, maxRequests: 20 }` 限制频率
- maxBodySize: 限制请求体大小（防撑爆）

### 8. AI 降级策略（fallback）
- 调用 AI → 失败 → 用预置的通用结果替代
- 先尽量用 AI 结果，不够的用 fallback 补

### 9. 数据库迁移（Migration）
- `supabase/migrations/` 下按时间命名的 .sql 文件
- 每次跑迁移：执行新文件，跳过已执行的文件
- 作用：数据库版本控制，多人开发时保持结构一致
- 初始迁移 = 完整建表，后续迁移 = ALTER 差异

### 10. NEXT_PUBLIC_ 前缀
- 带前缀：浏览器端可访问，嵌在前端代码里
- 不带前缀：仅服务端可用（如 SUPABASE_SERVICE_ROLE_KEY）

## 概念辨析
- React/Next.js/Vite 三者关系：React=UI 库，Next.js=框架，Vite=构建工具（LongCut 没用）
- Python 类比：React ≈ Flask，Next.js ≈ Django
- Next.js = React + 路由 + 后端能力

## 组件阅读实战：url-input.tsx
- 组件文件结构：import → interface → function → state → useEffect → handler → JSX
- HTML 原生标签（小写）vs 自定义组件（大写开头）
- div 默认垂直排列，className="flex" 改为水平
- 条件渲染：{条件 && <组件>}
- e.preventDefault() 阻止浏览器默认的页面刷新行为
- handleSubmit 三步：阻止刷新 → 验证 URL → 调用父组件函数

## 函数 Props 的本质
- JSX 是函数调用的语法糖：<UrlInput onSubmit={fn} /> 等价于 UrlInput({ onSubmit: fn })
- 子组件通知父组件的唯一方式：调用父组件传进来的函数
- 与 Python **kwargs 解包传参类似

## 乐观更新问题修复
- setIsUpdating(true) 在前面导致按钮先禁用，乐观更新效果被抵消
- 修复方案：去掉 isUpdating，直接乐观更新不显示加载

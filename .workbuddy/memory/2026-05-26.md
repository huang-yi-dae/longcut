## Stage 5.4 — Server Components vs Client Components

- Server Component：不加 `"use client"`，在服务端运行，无 Hook/事件/浏览器 API
- Client Component：加 `"use client"`，在浏览器运行，全功能
- 原则：能用 Server Component 就不用 Client Component
- 组件拆分策略：纯展示拆为 Server，只有交互的加 `"use client"`
- Server Component 可以 import Client Component，反过来不行

## Stage 5.6 — API Routes

- page.tsx 返回 HTML（页面），route.ts 返回 JSON（数据）
- GET/POST 是导出函数名而非关键字，是 Next.js 约定
- withSecurity = 安检门中间件，参数1=业务handler，参数2=预设安全等级
- AUTHENTICATED（全套+登录）/ PUBLIC（基础+免登录）
- 六道检查全在 withSecurity 内部自动完成

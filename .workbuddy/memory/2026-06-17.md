## 阶段 2 — MongoDB 面试题

### 1. Collection & Document 概念对照
- MongoDB Collection = PostgreSQL 表（Table）
- MongoDB Document = PostgreSQL 一行（Row）
- MongoDB Field = PostgreSQL 列（Column）
- 本质一样，叫法不同，Document 是 JSON 格式、字段可以不固定

### 2. 嵌入 vs 引用
- MongoDB 文档最大 16MB，一对多关系必须引用
- 嵌入：子数据直接作为父文档字段，适合一对一/一对少 + 数据总一起查
- 引用：单独建 Collection 用 ID 关联，适合一对多 + 数据独立查询
- LongCut 里 user_videos 就是引用模式（user_id 关联用户）

## 阶段 2 — Redis 面试题

### 3. Redis vs Memcache
- Redis：5 种数据类型 + 支持持久化 + 现在主流
- Memcache：只有字符串 KV + 重启全丢 + 老项目可能还在用
- KV = Key-Value（键值对），最简单的映射关系

### 4. Redis 过期删除策略
- 惰性删除：查的时候发现过期才删
- 定期删除：每 100ms 随机抽一批检查
- 内存淘汰：内存满了强制删
- 三种策略组合使用，不是立即删除

### 5. Redis 5 种数据类型
- String：单个值（get/set）
- List：队列/栈
- Set：不重复集合（去重）
- Hash：JSON 对象（hset/hget，省序列化）
- Sorted Set：带分数的集合（排行榜）
- 面试用 String + Hash 就够了，其他知道即可

## 阶段 2 — 数据库对比复习

### 6. 三大数据库综合对比
- PostgreSQL：关系型，存结构化核心数据（用户/视频/订单）
- MongoDB：文档型，存灵活多变的文档（日志/配置）
- Redis：内存缓存，高频访问数据加速（热门AI解析结果）
- 三者互补，不是替代关系
